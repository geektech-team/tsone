---
name: object-oriented-design-constraints
description: Use when designing, reviewing, refactoring, or implementing code that uses classes, interfaces, object-oriented design, OOP, UML class relationships, or SOLID principles.
---

1. 类关系必须明确，并以这 6 种关系为基础设计：依赖、关联、聚合、组合、继承和实现。
2. 应当满足面向对象设计的 SOLID 原则：单一职责原则、开闭原则、里氏替换原则、接口隔离原则、依赖倒置原则。
